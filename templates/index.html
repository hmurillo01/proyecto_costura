Prototipo - Gestión de Tareas textil

Problematica


En una fábrica textil, se presenta una problemática significativa relacionada con la asignación de tareas a sus costureras. Estas costureras son esenciales en el proceso de confección de prendas, pero la falta de un sistema eficiente de asignación de tareas está generando dificultades en la producción y tensiones en el equipo. Por ejemplo, consideremos el caso de una fábrica que necesita confeccionar una colección de camisas de alta demanda para la próxima temporada. Las costureras, responsables de coser las piezas de tela para ensamblar las camisas, enfrentan problemas para recibir instrucciones claras y específicas sobre cuántas camisas deben producir en cada turno y qué tipo de costuras o detalles deben priorizarse en cada una. Esta falta de claridad en la asignación de tareas provoca confusiones, retrasos en la producción y, en última instancia, afecta la calidad de las prendas. Algunas costureras pueden sentirse abrumadas por una carga de trabajo desigual, mientras que otras pueden tener períodos de inactividad, lo que conduce a un uso ineficiente de los recursos y a tensiones entre los miembros del equipo. Además, la falta de un sistema transparente y equitativo para asignar tareas puede generar resentimiento entre las costureras y la gerencia, lo que afecta negativamente el ambiente laboral y la motivación del personal. Es imperativo abordar esta problemática implementando un sistema de asignación de tareas más justo y eficiente, que garantice una distribución equitativa de la carga de trabajo y promueva una comunicación clara y abierta entre la gerencia y el equipo de costureras.



Requisitos Funcionales


Gestión de Tareas:


El sistema debe permitir a la fábrica crear, asignar, modificar y eliminar tareas para las costureras. Debe ser posible especificar la cantidad de prendas que se deben producir en cada tarea, así como los detalles específicos de la confección.


Asignación Tareas:


El sistema debe garantizar la asignación de tareas a las costureras

Debe permitir descripción de las tareas para brindar instrucciones detalladas.


Seguimiento de Tareas y Progreso:


El sistema debe permitir realizar un seguimiento del progreso de cada tarea, incluyendo el estado actual y el tiempo estimado de finalización.

Debe permitir descripción de las tareas para brindar instrucciones detalladas.





















Requisitos No Funcionales


Acceso y Seguridad:


Debe haber diferentes niveles de acceso para garantizar que solo las personas autorizadas puedan crear, asignar o modificar tareas.(Fase 2)

La información confidencial, como los datos personales de las costureras o los detalles de diseño de las prendas, debe estar protegida y accesible solo para aquellos con los permisos adecuados. (Fase 2)


Flexibilidad y Adaptabilidad:


El sistema debe ser lo suficientemente flexible como para adaptarse a cambios en la demanda de producción, cambios en los recursos disponibles o cambios en las preferencias de diseño de las prendas.

Debería permitir una fácil reasignación de tareas y ajustes en los plazos según sea necesario (Fase 2).
